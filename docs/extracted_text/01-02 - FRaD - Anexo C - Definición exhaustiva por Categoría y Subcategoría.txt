ANEXO C · Atributos Específicos por SubcategoríaAsignación, Obligatoriedad y Reglas de ValidaciónProyecto: LONDOR Enterprise SystemDocumento: FRaD · Módulo InventarioVersión: v1.0Estado: Definitivo – alineado con Anexo ARol: Anexo normativo de comportamiento de ficha dinámicaC.1 Objetivo del AnexoDefinir de forma exhaustiva, para cada subcategoría de joya, los:atributos aplicablesniveles de obligatoriedadreglas condicionalescriterios de validación funcionalEste anexo es el que convierte la clasificación en comportamiento real del sistema, gobernando:el wizard de altala ficha dinámicalas validaciones de guardadola coherencia del inventarioC.2 Principios de GobiernoNingún atributo se crea en este anexo (provienen del Anexo B)Toda subcategoría debe tener:atributos mínimos obligatoriosLas reglas deben poder validarse sin interpretación humanaSi un atributo no aplica, no se muestraNo se permiten fichas “incompletas funcionalmente”C.3 Tipología de atributosObligatorio (O): siempre requeridoOpcional (OP): visible pero no requeridoCondicional (C): obligatorio si se cumple condiciónNo aplicable (NA): no visibleC.4 Definición
